    覚えておくコマンドと意味（最小セット）¶

    この章でできるようになること¶

    • よく出るコマンドの「意味」を説明できる（丸暗記しない）
    • とくに add と commit の違いを言葉にできる
    • 「いま何が起きているか」を status で確認する癖がつく

    まずはこの3つ（最重要）¶

    git status（いまの状態を確認）¶

    git status

    何が起きるか: 変更ファイル、ステージ状況、次にやることのヒントが出ます。

    git add（コミットに入れる変更を選ぶ）¶

    git add <file>

    何が起きるか: 変更を ステージに載せます（「次のコミットに入れる」選別）。

    git commit（変更を履歴として確定）¶

    git commit -m "メッセージ"

    何が起きるか: ステージ上の変更を コミットとして確定し、履歴に残します。

    add と commit の違い（会話用の言い方）¶

    • add: 「次の提出物に入れるページを選んだ」
    • commit: 「提出して、提出番号（履歴）を発行した」

    add は「準備」、commit は「確定」です。

    覚えると会話が通じるコマンド（頻出）¶

    git diff（差分を見る）¶

    • ステージ前の差分:
    git diff
    • ステージ後の差分:
    git diff --staged

    git log（履歴を見る）¶

    git log --oneline --decorate --graph --all

    GitHubとセットで出てくるコマンド（最低限）¶

    git clone（リポジトリをコピーして持ってくる）¶

    git clone <repo-url>

    git pull（リモートの更新を取り込む：fetch + merge）¶

    git pull

    注意: pull は内部的に「取りに行く（fetch）」と「取り込む（merge）」が一緒に起きます。

    git push（自分の履歴をリモートへ送る）¶

    git push

    ブランチ関連（“概念がわかればOK”セット）¶

    git branch（ブランチ一覧）¶

    git branch

    git switch（ブランチを切り替える / 作る）¶

    git switch <branch>
git switch -c <new-branch>

    git merge（分岐を合流する）¶

    git merge <branch>

    cherry-pick（特定のコミットだけを別ブランチに持っていく）¶

    「この修正だけを release ブランチにも入れたい」のように、コミットを1つ（または複数）選んで取り込むときに使います。

    git cherry-pick <commit-sha>

    会話での言い方例:

    • 「このコミットだけをリリースに入れたいので cherry-pick して」
    • 「バックポート（backport）する」= 別のブランチにも同じ修正を入れる

    注意（ざっくり）:

    • cherry-pick は“履歴をコピーする”操作なので、取り込み先の状況によっては コンフリクトが起きます

    変更を取り消す（安全な2つだけ）¶

    ここで言う「戻す」は、履歴（コミット）を消すという意味ではなく、まずは

    • 作業中の変更を取り消す（ワークツリー）
    • “コミットに入れる選択”をやり直す（ステージ）

    という 安全寄りの2つだけに絞っています。

    git restore（作業中の変更を戻す）¶

    git restore <file>

    git restore --staged（ステージから外す：変更は残す）¶

    git restore --staged <file>

    補足:

    • 「コミットを取り消したい」系（revert / reset）は別トピックで整理します（ここより少し難しい）

    「checkout」って何？（いまは switch / restore が推奨）¶

    昔はブランチ切り替えもファイル復元も git checkout でやるのが一般的でした。
    今は Git が用途を分けたコマンドを用意していて、会話もそちらが増えています。

    • ブランチを切り替える: git switch
    • ファイルを戻す: git restore

    それでも checkout はまだ使われています（“非推奨で使えない”ではない）ので、意味だけ“翻訳”できればOKです。

    git checkout（ざっくり意味）¶

    • ブランチ切り替え（昔の書き方）:
    git checkout <branch>
    • ブランチ作成して切り替え（昔の書き方）:
    git checkout -b <new-branch>

    最後に：迷ったら status¶

    Git操作で不安になったら、まずこれです。

    git status
