# はじめに（Git / GitHub）

## この資料でできるようになること

- Git / GitHub の役割を切り分けて説明できる
- 会話で出る用語（commit / branch / PR / issue など）を誤解なく理解できる
- 最低限のコマンド（`status` / `add` / `commit` など）が「何をしているか」を言葉で説明できる

## 前提

- コマンドライン（ターミナル）を使える
- エディタは何でもよい（VS Code など）

## 用語（最小）

- **リポジトリ（repo）**: 履歴を含めてファイルを管理する箱
- **コミット**: 変更を「意味のある単位」で記録したスナップショット
- **ブランチ**: 履歴の分岐。安全に別作業をするための道筋
- **リモート**: GitHub など、ネット上のリポジトリ
