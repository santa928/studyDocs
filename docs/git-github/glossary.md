# 用語集（会話のため）

目的: 非エンジニアでも、会話でよく出る言葉を「誤解なく」理解するための最低限の辞書。

## Git（ローカルの概念）

- **リポジトリ（repository / repo）**
  - 履歴を含めてプロジェクトを管理する箱
- **コミット（commit）**
  - 変更を確定した履歴の単位（スナップショット）
- **ステージ（stage / index）**
  - 次のコミットに入れる変更を選ぶ場所
- **ワークツリー（working tree）**
  - いま編集している作業中のファイル群
- **ブランチ（branch）**
  - 履歴の分岐。別作業を安全に進めるための道筋
- **マージ（merge）**
  - 分岐した履歴を合流すること
- **コンフリクト（conflict）**
  - 同じ場所を別々に変更していて、自動で合流できない状態
- **HEAD**
  - 「いま見ている/作業している地点」を指す目印

## GitHub（共有と合意形成の概念）

- **リモート（remote）**
  - GitHub などネット上のリポジトリ
- **origin**
  - リモートにつける標準的な名前（慣習）
- **Pull Request（PR）**
  - 変更を取り込んでほしいという提案。差分確認とレビューの場
- **レビュー（review）**
  - PRへの確認。承認/指摘/質問など
- **マージ（merge）**
  - PR の変更を main などに取り込むこと（Git操作の merge と同じ言葉が使われる）
- **Issue**
  - 相談・タスク・バグを記録するチケット
- **ラベル（label）**
  - Issue/PR を分類するタグ
- **マイルストーン（milestone）**
  - 期限やリリース単位のまとめ

## 会話で混ざりやすい言葉

- **Git と GitHub**
  - Git: 履歴管理の仕組み（道具）
  - GitHub: 履歴を共有・レビューするサービス（場所）
- **push と pull**
  - push: 自分の履歴をリモートへ送る
  - pull: リモートの履歴を取り込む（実体は fetch + merge）
- **clone と pull**
  - clone: 初回に「丸ごと」持ってくる
  - pull: 2回目以降に「差分」を更新する
