# コミットを取り消す（revert / reset）

## この章でできるようになること

- 「コミットを取り消したい」相談で混乱しない
- revert と reset の違いを“会話レベル”で説明できる

## まず結論（会話のための最短）

- **共有後（GitHubにpushした後）**に「取り消し」たいなら、まずは **revert** の発想
- **手元だけ（まだ共有してない）**なら、reset でやり直す発想が出てくる

## `git revert`（取り消しコミットを“追加”する）

revert は「その変更を打ち消すコミットを追加する」方法です。  
履歴を消さないので、共有後でも比較的安全です。

## `git reset`（履歴を“戻す”）

reset は「履歴の先頭（HEAD）を過去に戻す」イメージです。  
やり直しには便利ですが、共有後に使うと他人と履歴がズレて混乱しがちです。

## 非エンジニア向けの合言葉

- 「もうGitHubに上がってるなら、revert 方向で相談」
- 「まだ自分の手元だけなら、reset でやり直せるかも」

具体的な手順はチーム運用次第なので、困ったらエンジニアに確認すればOKです（ここは無理に自走しなくて大丈夫）。
